# 新脚本远程核心制作流程

这份文档是给后续制作新脚本时直接照着走的短说明，尽量不用理解太多底层概念。

## 1. 先决定脚本用哪种发布方式

后台脚本管理里有 4 种模式：

1. 原始直装
2. 混淆直装
3. 验证增强
4. 远程核心

如果你希望脚本：

- 需要审批才能用
- 能单机绑定
- 后台能看使用情况
- 后续可以在服务器换逻辑

就选 `远程核心`。

## 2. 新脚本接入顺序

做一个新的远程核心脚本，按这个顺序：

1. 在脚本管理中创建脚本记录
2. 发布模式改成 `远程核心`
3. 打开“授权与运行时”页面
4. 在“远程核心”页先选中这个脚本
5. 新建一个模块版本
6. 再保存一个发布版本
7. 最后发布模块和版本

## 3. 模块怎么写

远程模块至少要导出：

```js
export async function bootstrap(context) {}
```

可选：

```js
export async function destroy(context) {}
export function getHealth() {}
```

## 4. 哪些东西放远程模块，哪些东西放运行参数

### 适合放模块里的

- 业务主流程
- 页面操作逻辑
- 按钮点击、队列执行、扫描流程

### 适合放运行参数里的

- 选择器
- 文案
- 学习速度
- 等待时间
- 功能开关
- 站点兼容规则

## 5. 后台每个区块是干什么的

### 运行入口

填脚本去哪里连授权服务和远程模块服务。

### 远程核心

管理这条脚本当前发布的版本和模块。

### 授权审批

看用户提交的使用申请，决定批不批。

### 已发授权

看已经批准过的授权，能停用、重置设备。

### 设备与在线

看授权绑到了哪台机器、哪些页面还在线。

### 最近记录

看用户有没有申请、有没有激活、页面有没有启动和报错。

## 6. 正式上线前要检查什么

至少检查：

1. 运行时地址必须是 HTTPS
2. 申请提交正常
3. 审批通过后脚本能自动继续激活
4. 激活后能看到设备绑定
5. 激活后能看到在线会话
6. 脚本页面关闭后，会话会在超时后结束

## 补充说明（2026-05-11）

现在后台已经支持更适合小白的远程核心接入流程：

1. 先在“脚本管理”里把脚本创建出来，并把交付方案切成“远程热更新”
2. 上传一个当前能跑的完整脚本版本
3. 打开“授权与运行时”页，进入“远程核心”
4. 选中对应脚本后，直接点“一键生成草稿”
5. 系统会自动基于最新上传版本生成远程模块代码骨架，并同时给出默认 `runtimeConfig`
6. 优先把选择器、文案、节奏参数放进 `runtimeConfig`
7. 保存模块后，再保存 Manifest，最后再发布

如果只是第一次接入，不要一开始就硬拆很多模块。先让完整逻辑通过草稿骨架跑起来，再逐步抽离容易变化的部分。
