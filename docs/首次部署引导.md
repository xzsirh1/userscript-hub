# 首次部署引导

新部署的空数据库不会再自动创建 `admin/admin` 管理员，也不会默认使用个人化站点名称。首次访问平台时，系统会进入 `/setup` 初始化页。

## 初始化会设置什么

- 平台名称：显示在前台、后台侧边栏和浏览器标题中。
- 平台简称：未上传 favicon 时用于默认图标。
- 平台描述：用于前台说明和站点配置。
- 管理员用户名：初始化后用于登录后台。
- 管理员密码：至少 8 位，初始化后请妥善保存。

## 初始化后的行为

- 初始化完成后会写入 `site_config.setup_completed = 1`。
- 系统会创建第一位管理员账号，并自动进入后台。
- `/setup` 会关闭，再访问会跳回 `/admin/login`。
- 已存在管理员的旧数据库会被视为已初始化，不会被强制打断。

## 重置初始化

不建议在生产环境手动重置初始化状态。若确实要重新初始化，应先备份数据库和上传文件，然后清空或替换 `server/data/database.db`，再重新启动服务。

## 开源部署建议

- 生产环境必须设置自己的 `JWT_SECRET`。
- 初始化后立刻确认后台“网站设置”里的平台名称、公告和图标。
- 不要在公开文档中保留真实服务器账号、密码、域名或个人品牌名。
