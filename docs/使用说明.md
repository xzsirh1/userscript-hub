# 使用说明

这份文档只保留日常使用说明，不再混入某一次更新包的临时信息。

## 角色入口

- 普通访问者：前台首页、脚本列表、插件列表、小程序列表
- 管理员：`/admin` 登录后台后进行资源管理、备份、同步、更新

## 后台主要功能

### 资源管理

- 脚本管理：上传版本、下载原版、设置私密访问、开启混淆
- 插件管理：上传插件文件，维护版本和浏览器类型
- 程序管理：上传小程序文件，维护版本、描述、推荐状态
- 分类管理：给脚本分组，影响前台列表展示

### 站点设置

- 修改站点标题、公告、主题信息
- 修改管理员密码
- 导出当前数据和上传文件
- 导入备份包恢复站点

### 数据同步

- 配置同步节点后，可在多个站点之间拉取数据
- 首次接入或历史数据不完整时，优先执行一次全量同步
- 涉及下载回源能力时，源站和拉取站要保持同版本

### 系统更新

- 在后台上传更新包进行在线更新
- 更新前系统会先备份代码
- Docker 和 PM2 都要使用包含 `client/dist` 的标准更新包

## 前台常见操作

### 脚本下载

- 公开脚本可直接安装或下载
- 私密脚本按配置要求输入密码，或仅管理员可访问
- 管理员下载默认取原版，普通用户可能取混淆版

### 插件和小程序下载

- 从对应列表页直接下载
- 下载次数会进入后台统计

## 备份与恢复

### 导出

- 入口：后台 `网站设置`
- 导出内容：数据库 + `uploads` 文件
- 导出完成后浏览器会直接开始下载 ZIP 包

### 导入

- 导入会覆盖当前业务数据和上传文件
- 导入前建议先再导出一次当前站点，留一份回滚包
- Docker 环境导入后，如果下载异常，优先确认文件是否真的恢复到了实际使用的 `uploads` 目录

## 同步与下载排查

出现“页面有记录但下载提示文件不存在”时，优先按下面顺序检查：

1. 本地站点 `uploads` 目录是否存在对应文件
2. 备份导入后文件是否真的恢复到运行目录
3. 同步节点是否已全量同步过历史数据
4. 源站与拉取站是否升级到同一版本

## 常见建议

- 部署完成后立刻修改默认管理员密码
- 更新包统一用 `node scripts/create-update.js` 生成
- 修改更新逻辑后，先验证构建，再验证 ZIP 结构，再上线
- 遇到 Docker 下载异常，优先看容器日志中的 `[download]` 关键字

### 3. 重启失败

**问题**: 服务重启失败
**解决**:
```bash
# PM2 环境
pm2 list  # 查看进程名
pm2 restart userscript-hub  # 手动重启

# Docker 环境
docker ps  # 查看容器名
docker restart userscript-hub  # 手动重启
```

### 4. 页面 404

**问题**: 访问 `/programs` 或 `/admin/programs` 显示 404
**解决**:
- 清除浏览器缓存（Ctrl+F5）
- 检查前端文件是否正确部署
- 查看浏览器控制台错误信息

### 5. 数据库错误

**问题**: 提示数据库表不存在
**解决**:
- 重启服务会自动创建 programs 表
- 如果仍然失败，手动执行 SQL：
```bash
sqlite3 server/data/database.db < create_programs_table.sql
```

## 📞 技术支持

如遇到问题：

1. 查看服务器日志
   ```bash
   # PM2
   pm2 logs userscript-hub
   
   # Docker
   docker logs userscript-hub
   ```

2. 查看浏览器控制台（F12）

3. 检查数据库是否正常
   ```bash
   sqlite3 server/data/database.db "SELECT * FROM programs;"
   ```

## 🔄 回滚方法

如果更新后出现问题：

1. 进入"系统更新"页面
2. 在"代码备份"列表中找到更新前的备份
3. 点击"回滚"按钮
4. 等待恢复完成
5. 手动重启服务
