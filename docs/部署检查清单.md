# 部署检查清单

这份清单用于每次发布、在线更新或手工部署前后的通用检查，不绑定任何个人服务器或历史版本。

## 更新前

- [ ] 站点当前可以正常访问。
- [ ] 管理员账号可以登录。
- [ ] 已备份数据库和上传文件。
- [ ] 磁盘空间充足。
- [ ] Docker / PM2 运行状态正常。
- [ ] 更新包来自 `build/archives/`，并由 `node scripts/create-update.js` 生成。
- [ ] ZIP 内包含 `server/src`、`server/package.json`、`client/src`、`client/dist`、`scripts`、`docs`。

## 更新中

- [ ] 在线更新：登录后台，进入系统更新，上传最新 ZIP 包并等待更新完成。
- [ ] 手工更新：解压更新包，按 Docker 或 PM2 部署方式覆盖后端源码与前端构建产物。
- [ ] 重启服务并观察日志。

## 更新后

- [ ] 首页正常打开。
- [ ] 后台正常打开并可登录。
- [ ] `/api/health` 返回正常。
- [ ] 脚本、插件、小程序下载正常。
- [ ] 授权与远程核心关键流程正常。
- [ ] 数据备份和同步功能正常。

## 常用命令

- PM2: `pm2 logs userscript-hub`
- Docker: `docker logs userscript-hub`
- 健康检查: `curl http://127.0.0.1:3000/api/health`
